# Igor in the Pi — Roadmap

## North Star

Build a high-leverage AI operating system that helps Dima:

1. Build products loved by millions
2. Become an elite trader with disciplined systems
3. Compound execution speed without sacrificing quality

---

## Phase 1 — Foundation (Now)

- [x] Core identity and mission files
- [x] Repo scaffold and starter playbooks
- [ ] Long-term memory curation workflow (`MEMORY.md` cadence)
- [ ] Standard daily/weekly review routine
- [ ] Backup + restore procedure documented

## Phase 2 — Product Execution System

- [ ] Product idea intake template
- [ ] Prioritization scorecard (impact × confidence × effort)
- [ ] Experiment tracker (hypothesis, metric, result)
- [ ] Launch checklist and post-launch review format
- [ ] Feedback ingestion loop from channels

## Phase 3 — Trading Command Center

- [ ] Trading plan template (setups, invalidation, sizing)
- [ ] Risk guardrails codified (max loss/day, max risk/trade)
- [ ] Journal schema (pre-trade, execution, postmortem)
- [ ] Weekly performance report template
- [ ] Mistake taxonomy + anti-pattern playbook

## Phase 4 — Automation & Reliability

- [ ] Reminder library (cron templates)
- [ ] Heartbeat task packs (quiet/active modes)
- [ ] Health checks for gateway/channels
- [ ] Alerting thresholds (urgent vs informational)
- [ ] Recovery runbook for auth/connectivity failures

## Phase 5 — Scaling Intelligence

- [ ] Domain-specific skills library curation
- [ ] Reusable prompt kits for recurring work
- [ ] Multi-agent delegation patterns
- [ ] KPI dashboard for execution velocity
- [ ] Quarterly operating-system refactor

---

## Operating Rhythm

- **Daily:** top priorities, risk checks, quick journal
- **Weekly:** review outcomes, update systems, prune noise
- **Monthly:** strategic reset + roadmap update

## Success Metrics (Initial)

- Product: cycle time from idea → shipped experiment
- Trading: rule adherence %, max drawdown control, consistency
- Ops: % of critical tasks automated, incident recovery time
